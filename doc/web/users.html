Here is a list of companies and people that use or have used MLton. If you would like to add something to the list, please send mail to MLton@mlton.org with a brief description and a link.

Commercial

  • Hardcore Processing uses MLton as a crosscompiler from Linux to Windows for graphics and game software.
    • CEX3D Converter, a conversion program for 3D objects.
    • Interactive Showreel, which contains a crossplatform GUI-toolkit and a realtime renderer for a subset of RenderMan written in Standard ML.
    • various games
  • PolySpace Technologies uses MLton for their product that detects runtime errors in embedded systems based on abstract interpretation.
  • Sourcelight Technologies uses MLton internally for prototyping and for processing databases as part of their system that makes personalized movie recommendations.

Non-commercial

  • The Concert Project uses MLton to build some fo its front-end software.
  • Simon Helsen uses MLton to develop MLOPE, an offline partial evaluator for full Standard ML.
  • Tom Murphy uses MLton to implement replacements for standard network services (fingerd, ftpd, identd, ...) and to compile his entries (2001, 2002) for the annual ICFP programming contest.
  • Michael Neumann uses MLton to build Guugelhupf, a simple search engine.

Papers

Here are some papers that others have written that present projects that have used MLton or comparisons of other systems to MLton.
  • Automatic Code Generation from Coloured Petri Nets for an Access Control System
  • The HiPE/x86 Erlang Compiler: System Description and Performance Evaluation
    Describes a native x86 Erlang compiler and a comparison of many different native x86 compilers (including MLton) and their register usage and call stack implementations.
  • Programming With Recursion Schemes
    Describes a programming technique for data abstraction, along with benchmarks of MLton and other SML compilers.
  • Type-Preserving Garbage Collectors
    Shows how to generate a strongly typed garbage collector as part of a program, using MLton as a practical demonstration.
