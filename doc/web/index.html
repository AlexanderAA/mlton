What is MLton?

MLton is a whole-program optimizing Standard ML (SML) compiler.

Features of MLton

  • Runs on a variety of platforms.
    • X86: Linux, FreeBSD, or Cygwin/Windows
    • Sparc: SunOS
  • Generates standalone executables with excellent running times
  • SML 97 compliant, with a complete basis library (except for use) matching the 2002 basis library specification
  • Untagged native integers and unboxed native arrays
  • Fast C FFI
  • Source-level profiling of both time and allocation
  • Multiple garbage collection strategies (copying, mark compact, generational)
  • Supports large amounts of memory and large arrays
  • Fast IntInf based on the GNU multiprecision library
  • Libraries for continuations, interval timers, random numbers, resource limits, resource usage, signal handlers, system logging, threads, heap save and restore.

What's new?

Version 20030312 is the latest public release. For a complete list of changes and bug fixes since the last release, see the changelog. Notable changes since the last public release, 20020923, include:
  • Added source-level profiling of both time and allocation. mlprof can also produce source-function call graphs.
  • Updated basis library to 2002 specification. To obtain the old library, compile with -basis 1997.
  • Added many modules to basis library: BinPrimIO, GenericSock, ImperativeIO, INetSock, NetHostDB, NetProtDB, NetServDB, Socket, StreamIO, TextPrimIO, UnixSock.
  • Completed implementation of IntInf and OS.IO.

How do you pronounce MLton?

MLton is pronounced in two syllables, with stress on the first syllable. The first syllable sounds like the word mill (as in "steel mill"), the second like the word tin (as in "cookie tin").
